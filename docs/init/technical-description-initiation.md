# Техническое описание этапа Инициация

## 1. Цель документа

Зафиксировать техническое состояние проекта `Scrider` на этапе Инициации и определить базу для перехода к планированию и реализации.

## 2. Контекст

`Scrider` — WYSIWYG-редактор с акцентом на академические тексты, формулы и структурированный контент.

На момент Инициации подтверждена архитектурная модель:

- `@scrider/editor-core` — данные и чистые функции (`State -> State`);
- `@scrider/editor-react` — интеграция с DOM/React и `contentEditable`;
- `demo/` — стенд на `Vite` для проверки интеграционных сценариев.

## 3. Технические решения, принятые на старте

## 3.1 Архитектура

- разделение логики редактирования и фреймворк-слоя;
- отсутствие DOM-зависимостей в ядре;
- возможность тестирования ядра без браузера;
- расширение функциональности через registry/plugin-подход.

## 3.2 Механизм редактирования

- в UI-слое используется `contentEditable` как нативный механизм браузерного ввода;
- браузер обрабатывает курсор/выделение/ввод, редактор синхронизирует это с собственной моделью данных.

## 3.3 Технологическая база

- язык: TypeScript;
- UI: React 18+;
- dev/runtime стенд: Vite;
- тесты: Vitest;
- форматирование/качество кода: ESLint, Prettier.

## 4. Технические ограничения и зависимости

## 4.1 Ограничения

- поведение `contentEditable` может отличаться между браузерными движками;
- браузерный слой требует дополнительной нормализации DOM-событий;
- часть функций (рендер, взаимодействие с DOM) доступна только в браузере.

## 4.2 Внешние зависимости

- `@scrider/delta` — модель операций и преобразований контента;
- `@scrider/formatter` — конвертация Delta в текстовые/разметочные представления;
- React и экосистема сборки для UI-обертки.
