# Резюме проекта (этап Инициация)

## 1. О проекте

- Название: `Scrider`
- Тип продукта: WYSIWYG-редактор для академических и технических текстов
- Формат поставки: библиотека с framework-agnostic core и React-оберткой

## 2. Назначение

`Scrider` создается как редактор, ориентированный на академический контент:

- ввод и форматирование текста в режиме WYSIWYG;
- поддержка математических формул (LaTeX);
- работа с таблицами, списками, блоками кода, диаграммами и медиа;
- экспорт в пользовательские форматы (PDF, DOCX, HTML, Markdown, Plain Text, Delta JSON).

## 3. Техническая рамка

- Архитектура: `Framework-Agnostic Core + Framework Wrappers`
- Ядро: `@scrider/editor-core` (TypeScript, без DOM-зависимостей)
- UI-слой: `@scrider/editor-react` (React + DOM + `contentEditable`)
- Демо-стенд: `demo/` на `Vite`

## 4. Результат этапа Инициация

Этап считается закрытым при наличии базового комплекта документов:

- техническое описание этапа Инициация;
- первичный scope MVP (`in/out`);
- минимальный набор user stories и use cases с критериями приемки;
- первичный перечень нефункциональных требований;
- стартовый реестр рисков;
- трассируемость требований и проверок.

## 5. Сроки, ресурсы и роли

### Сроки

- ориентир по защите проектов в текущем семестре: `15–24 мая`;
- по каждому этапу предусмотрены видеовстречи (участие опционально).

### Ресурсы и рабочая среда

- проект ведется через задачи на платформе `Проектива`;
- изменения фиксируются в репозитории `GitHub` и должны соответствовать плану выполнения;
- для защиты ожидаются: рабочий MVP, наглядный пул выполненных работ, демонстрация командной работы;
- техническая среда по `README`: `TypeScript`, `React 18+`, `Vite`, `Vitest`, `ESLint`, `pnpm`, `Node.js 18+`.

### Роли и процесс выполнения

- задачи по этапам назначаются исполнителям в `Проективе`;
- исполнитель закрывает задачу с одновременным обогащением Git-репозитория;
- командная работа подтверждается совокупностью артефактов (доска задач + репозиторий + демонстрация).
