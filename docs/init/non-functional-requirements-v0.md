# Нефункциональные требования v0

## 1. Назначение

Первичный набор нефункциональных требований для `Scrider` на базе текущей архитектуры и технологического стека.

## 2. Совместимость

- браузерный слой должен работать в актуальных версиях современных браузеров (Chrome/Chromium, Edge, Firefox, Safari);
- поведение `contentEditable` должно быть функционально приемлемым в целевых браузерах при основных сценариях редактирования.
- интерфейс редактора должен корректно адаптироваться под разные размеры контейнера, включая мобильные экраны;

## 3. Архитектурная целостность

- `@scrider/editor-core` не содержит прямых зависимостей от DOM;
- функции ядра реализуются как чистые трансформации состояния;
- браузерно-зависимая логика выносится в слой интеграции с DOM.

## 4. Тестируемость

- критичные операции ядра покрываются unit-тестами без браузера;
- UI-сценарии редактирования покрываются интеграционными тестами с DOM;
- проверяется стабильность базовых операций (ввод, форматирование, undo/redo).

## 5. Расширяемость

- новая функциональность должна подключаться через существующие registry/plugin-механизмы;
- добавление расширений не должно требовать изменения базовых контрактов core без отдельного согласования.

## 6. Экспорт и целостность данных

- экспорт не должен терять весь контент и базовую структуру документа;
- преобразования между внутренним форматом и внешними представлениями должны быть детерминированными для базовых кейсов.

## 7. Ограничения v0

- метрики производительности, SLA и количественные пороги будут уточнены на этапе планирования;
- требования по безопасности и правам доступа фиксируются отдельно, если появятся в scope.
