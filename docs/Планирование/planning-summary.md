# Итоги планирования (этап 2)

## 1. Что зафиксировано по итогам этапа

- Планирование разделено на 2 части: общее планирование и планирование реализации продукта.
- Подтверждена этапная структура WBS: `Инициация -> Планирование -> Реализация -> Закрытие`.
- Зафиксированы ключевые задачи этапа `Планирование`:
  - создание `Product Roadmap`;
  - разработка `UI/UX` дизайна и прототипов;
  - планирование технической архитектуры;
  - формирование бэклога разработки;
  - утверждение плана проекта.
- Подтверждена операционная связка выполнения: задачи в `Проективе` <-> изменения в `GitHub`.
- Контрольная дата защиты проекта: ориентир `15–24 мая 2026`.

## 2. Что принято для дальнейшей работы

- В качестве базовой модели трудозатрат используется WBS разработки с долями:
  - анализ и проектирование — `35%`;
  - разработка и реализация — `20%`;
  - тестирование и QA — `30%`;
  - внедрение и развертывание — `10%`;
  - эксплуатация и сопровождение — `5%` (для MVP не критично).
- Для управления загрузкой используются две проекции:
  - по этапам проекта;
  - по участникам/ролям.

## 3. Артефакты этапа планирования

- `docs/planning-summary.md` — краткие итоги планирования.
- `docs/workload-estimation-by-stages-and-participants.md` — оценка загрузки по этапам и участникам.
